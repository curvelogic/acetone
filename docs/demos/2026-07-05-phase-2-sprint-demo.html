7/12 · the lab earned its keep

Two real bugs the small tests couldn't see

1 · The executor didn't scale

The first full-scale run took 30 s, 21 s and 147 s on the multi-hop queries — the graph adapter scanned the whole edge set on every expansion, O(nodes·edges) over a MATCH. Indexing node/label/adjacency at construction dropped the worst case from 147 s to ~1 s; the reviewer proved result-equivalence by a TCK A/B (identical 1,372, zero mismatches).

2 · A node's identity wasn't queryable

The flagship var-length query silently matched nothing: it filtered on a key property, and the adapter never re-exposed key values. That was a latent bug in the already-merged CLI too. Fixed at the root — keys are now filterable and returnable under their schema names.

A realistic graph is a test the unit suite can't write. Both fixes came out of running one.

8/12 · gate evidence · exit criterion 3

AT <ref> — history is queryable

Whole-query --at <ref> and clause-group MATCH … AT <ref> both work over the Phase 1 commit graph. A single query can address several versions at once.

MATCH (h:Host) AT main~5
MATCH (h)-[:HAS_CERT]->(c) AT main
RETURN h.hostname, c.not_after

Cross-version identity is sound — and it's a free consequence of Load-Bearing Invariant #3. A node's identity is its (label, key), which is version-stable; only its property values are a snapshot. So re-anchoring an old node in a later clause walks the later topology deterministically. The reviewer verified this directly.

9/12 · what review caught

The gate did its job

Every PR was reviewed by a fresh subagent with no implementation context. Real defects the implementation and its own tests had missed:

10/12 · milestone security review

No blocker-class findings — gate security-ready

A dedicated security pass (strongest tier, 30+ adversarial probes, cargo deny) over the whole phase diff. Verified clean:

  • Parser — bounded (depth 64 parse / 256 AST) and panic-free under 100k-deep nesting, 5 MB literals, adversarial UTF-8, on a 2 MiB stack.
  • Arithmetic — overflow, /0, %0, i64::MIN / -1, slice bounds all return errors, not panics.
  • Ref/path injection — refspecs validated through gix; no filesystem escape, no process spawn; the REPL never shells out.
  • Dependencies — clean; the spike's decypher is genuinely workspace-excluded.

One open risk carried forward: the query engine has no resource governor — an unbounded MATCH (a)-[*]->(b) or range(0, 1e10) can exhaust CPU/memory. Self-inflicted for the CLI; the real risk is the library-embedding case. Filed as acetone-iq6, the top hardening priority. Two cheap escape-sanitisation gaps were fixed in #39 with mutation-proven tests.
