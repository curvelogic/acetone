03 / 09

The flagship

A mutating source, synced as history.

Each sync of the source is an import commit carrying its provenance. Re-syncing the same graph is a detected no-op. The diff between two syncs is the change report.

$ acetone import ndjson snap2.ndjson --label Host
imported 4 node(s) and 0 edge(s) onto the current branch; commit 39fe364…
$ acetone diff 9e92004… 39fe364…
+ node "Host" ["new1"] ← a host appeared
~ node "Host" ["web1"] ← web1 was re-imaged
$ acetone import ndjson snap3.ndjson --label Host # same graph
source unchanged; nothing imported
04 / 09

Declared indexes

An equality that seeks, not scans.

Declare an index on a property and the binder's IndexSeek hint is now honoured by the executor: an O(matches) lookup instead of scanning the whole label. On the 44k-node lab graph (--scale 20000), MATCH (h:Host {os:'debian'}):

INDEXSEEK

~14 ms

the declared host_os index

LABELSCAN + FILTER

~69 ms

the whole Host population

SPEEDUP

4.9×

same rows, proven by a parity test

  • Maintained transactionally — a delta per touched node; reindex rebuilds to identical roots (Invariant #5). null- and NaN-blind.
  • The seek returns exactly the scan's rows — numeric 3 = 3.0, the f64 2⁵³ boundary, and Bytes/temporal representation were each caught by the review and fixed.
05 / 09

Decisions taken

Two ADRs, both mid-phase.

ADR-0021 · import interface

Interface in acetone-graph (no seventh crate); format extractors + csv/serde_json/sha2 confined to the CLI. Authoritative-replace apply; source hash = sha256 of the raw bytes; no-op = workspace-vs-HEAD.

ADR-0022 · IndexSeek execution

Seek runs over an in-memory value map keyed by the runtime representation (a lazy store-backed seek is the deferred scalability win). Numeric equality probes both Int and Float buckets. Equality only — IndexRange deferred.

Both were decided by ADR so the work could proceed, and are flagged here for your retrospective review.

06 / 09

The review gate earned its keep

A real, shipping-blocking bug on every feature PR.

Every PR passed a fresh-subagent adversarial review with no implementation context. The gate did not rubber-stamp — it caught genuine defects, several of the silent-wrong-results class, each fixed and re-reviewed before merge.

#65
Import — provenance trailers validated after save left the workspace dirty and stranded the caller under --branch. → validate up front.
#67
IndexSeek — three rounds, three silent subset bugs: 3 = 3.0 cross-type, the f64 2⁵³ precision boundary, and Bytes/temporal keyed off the runtime representation. All fixed.
#68
Export — path traversal via an attacker-controlled label name, and a rel type spanning two label pairs silently producing wrong roots. → sanitise + reject.
#69
gc — consolidation can't see other worktrees' private refs, so a cross-worktree gc could prune their uncommitted work. → refuse while linked worktrees exist.

Gate evidence & security

Exit criteria met; security gate ready.

  • Index faster than scan — 4.9× on the lab graph, with a parity test proving the seek's rows equal the scan's.
  • fsck — index consistency and a history-independence spot-check; each category has a seeded-corruption test.
