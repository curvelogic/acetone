03 / 09

The workbench loop

Edit a version-controlled graph
the way you edit code.

  • Schema first. Natural-key identity is mandatory (Invariant #3), so a label declares its key before Cypher can create it.
  • Write, then read back. A fresh process reads what the write persisted — it's on disk, in the workspace, not in memory.
  • Atomic. The workspace advances by compare-and-swap; a failed write leaves it untouched.
  • Commit. The workspace manifest becomes a real git commit — git log, clone and push all work natively.
$ acetone declare-label Host --key name --require os
$ acetone query "CREATE (:Host {name:'web-01', os:'debian'})
-[:RUNS]->(:Software {name:'nginx'})"
2 nodes created, 1 relationship created
$ acetone query "MATCH (h:Host)-[:RUNS]->(s) RETURN h.os, s.name"
debian | nginx # fresh process — persisted
$ acetone commit -m "seed the estate"
committed 35dbe32…
04 / 09

Decisions · for your retrospective

Two ADRs, both about the workspace.

Building save on the workspace forced two storage decisions — made by ADR so work proceeded, flagged here.

ADR-0014 · amends 0010

Worktrees are first-class

The writer lock and the workspace move to a per-worktree ref (refs/worktree/acetone/workspace). Two git worktrees of one clone now edit different branches independently — the git-native way to have two branches open at once. A probe confirmed gix 0.85 resolves the namespace per-worktree; a pre-rjf repo migrates on first write.

ADR-0015 · amends 0010/0014

Uncommitted state is gc-durable

The workspace ref points at a {manifest, chunks/} tree, so git's reachability keeps the workspace's chunks. An uncommitted workspace now survives git gc --prune=now — closing the "commit before gc" caveat. Local-only ref plumbing; no format bump. Incremental anchoring deferred.

The review gate, working

Every PR met a fresh adversarial reviewer.
Four had blockers. All fixed.

Each merge gate was a fresh subagent with no implementation context, strongest model tier. The blockers it caught were real correctness bugs a test hadn't yet reached:

#41 · CREATE

Attaching labels/props to an already-bound node was silently dropped — now a bind-time error.

#42 · SET

A later write clobbered an AT <ref> time-travel snapshot — fixed with an override-only refresh.

#43 · DELETE

The standard DELETE r, a idiom was spuriously rejected — connectivity check deferred to clause end.

#49 · constraints

Constraints ignored same-transaction deletes, false-rejecting the rekey path — and the fix surfaced a deeper delete-before-put ordering bug.

The other six PRs were accepted with no blockers — each verified by targeted experiments (independent MERGE execution, encoder tracing, live worktree/gc probes, self-loop & parallel-edge rekey).
