3/9 · the format

Node identity has exactly one byte form, everywhere

A node key — (primary label, key tuple) — encodes as a single memcomparable list element, and those exact bytes appear standalone as the nodes-map key and embedded inside edge and index keys (ADR-0008). Identity comparisons are byte comparisons in every map; label scans are prefix scans.

("Host", "web1")  →  0c 06 48 6f 73 74 … fb 06 77 65 62 31 … fb 00
nodes map key         └── the same bytes ──┘
edges_fwd key   [src node key][type][dst node key][disc]
  • Key properties live only in the key — a record that disagrees with its key is unrepresentable (Invariant 3 enforced structurally).
  • The manifest is version-first: [format_version, body], so any future reader identifies a newer repository before parsing a byte of body.
  • All of it golden-pinned: changing any byte is a declared format_version bump, not an accident.

4/9 · the repository

An acetone repo is a bare git repo you can read with git

a commit, in git's eyes

commit fe0083c…
├── README.md      (human summary)
├── manifest       (the version record)
└── chunks/        (anchors every chunk)

git log, git show, hosting UIs, git fsck --strict: all just work. Trailers carry provenance.

the moving parts (ADR-0010)

  • Workspace = manifest blob at refs/acetone/workspaces/<name>, advanced by compare-and-swap — a lost race is a typed error, never a lost write.
  • Single-writer lock held for a whole transaction; readers are pinned to immutable manifests (MVCC for free).
  • Real git HEAD is the checked-out ref — clones see the right branch.

Proven, not claimed: a committed 500-node version survives git gc --prune=now --aggressive and a cold reopen — commits anchor the complete chunk set of every map, so whole versions travel with clone/push/fetch.

5/9 · exit evidence

The e2e script, and a real GitHub round trip

scripts/phase1-e2e.sh runs in CI on every push: init → plumbing writes → commit with trailer → branch → mutate → root-level diff (diverged branches differ at <branch>:manifest) → acetone fsck clean → native git interop → gc survival.

Run manually against github.com/curvelogic/acetone-phase1-e2e (private, created as evidence):

  • pushed main + feature; bare-cloned back; git fsck --strict clean on the clone;
  • workspace recreated in the clone with one line of plain git plumbing (git update-ref … main:manifest);
  • acetone served node data, history and a clean fsck from the clone.

Spec §3.5, demonstrated end to end: “a remote need not know acetone exists.”

Property suites green across the workspace: history independence, encoding order/canonicity, map-level merge determinism — plus an experimental witness for Invariant 2 (5,000 records × every single-bit flip: zero aliasing).

6/9 · the hard two

Pack-on-write productionised; fsck born paranoid

pack-on-write (#21, ADR-0011)

  • Predecessor recording as a pure observer on apply_batch — output bit-identical, proven by test.
  • Hand-rolled git delta encoder + v2 pack/idx writers; every delta round-trip-verified against exact bytes before use.
  • gc consolidation: bases-first topological order, chain caps, fsync-before-prune, prune gated on the written pack's own contents — a planning bug fails loudly, never deletes data.
  • Phase 0 measured the win at 7.1× retention; the machinery now exists to collect it.

fsck (#22, ADR-0012)

  • Verifies every version reachable from workspaces, branches and tags: manifest decode, chunk reachability, prolly structure at claimed positions.
  • MISSING and CORRUPT distinguished; every finding names its chunk.
  • Advisory tier for later-phase semantics (edge symmetry, annotated tags) — incompleteness is declared, never silent.
  • Four review rounds; two proven false-cleans found and killed before merge (next slide).

8/9 · decisions & risks

Five ADRs on record; open risks are beaded and bounded

decisions taken (made, not deferred)

  • 0008 map layouts + manifest encoding · 0010 workspace refs, writer lock, HEAD semantics · 0011 pack-on-write consolidation (includes a deliberate spec §3.1 amendment) · 0012 fsck taxonomy · 0009 subagent model tiers (yours).

open risks (none blocker-class)

  • P2 write_ref create-CAS accepts a value-equal no-op (acetone-0ej) · fsck can't run if the default workspace manifest itself is the damaged object (acetone-zhp) · fsck tag-peeling and cross-version dedup (acetone-8t3/7fe).
  • Format-level gap, documented: uncommitted workspaces don't survive a foreign git gc until acetone-gc protects them (ADR-0010).
  • P3s: checkout two-ref atomicity, library no-change-commit guard, consolidation scaling, anchor-completeness check, and friends — all beaded.
