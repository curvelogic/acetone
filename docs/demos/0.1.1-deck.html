Errors that help

The storage engine's discipline, in the messages.

Keys read like keys, every error has a place, and a typo is met with a suggestion instead of a dead end.

BEFORE
CREATE of "Topic" "[String(\"x\")]" conflicts with an
existing node; identity must be unique (use MERGE to upsert)
AFTER
CREATE of "Topic" ["x"] conflicts with an existing node;
CREATE always makes a new node … MERGE each node on its own
before MERGEing a relationship between them
BEFORE
division by zero at bytes 7..12
unknown label 'Hsot' (declared labels come from the schema map)
AFTER
line 1, column 8: division by zero
unknown label "Hsot" (not declared in the schema) — did you mean "Host"?

All attacker-writable text reaching the terminal is escaped — verified end to end by the security review.

04 / 11

A CLI that reads like one

Grouped, forgiving, and honest about git.

HELP

Grouped by role, with the git relationship

acetone --help is one line per command, grouped Everyday / Schema / Data & query / Maintenance / Plumbing — and states which commands you use instead of git, which either works, and which are plain git (transport). Since an acetone version is a git commit, that was the hidden question.

FORGIVING

Aliases, prefixes, discovery

cypher aliases query; a unique prefix resolves (acetone st → status); import/export take a consistent --format; and acetone from any subdirectory finds the enclosing repo, git -C-style — while preserving the reduced-trust config-isolation boundary (ADR-0034).

05 / 11

See it and script it

Read the schema. Script the graph.

acetone schema

The one thing you couldn't see

Schema is the load-bearing part of the model — identity keys — yet 0.1.0 only showed a count. acetone schema [--at <ref>] now lists labels + keys, relationship types and indexes for any version, without checking it out.

--json

Machine output, honestly labelled

--json on status, log, branch, diff, list-nodes, get-node and schema makes the read side scriptable. The shape is deliberately not frozen — it may change before 0.2, when the acetone-core facade can own it.

$ acetone schema
Labels
"Host" key ("hostname") required ("os")
Relationship types
"RUNS"
$ acetone status --json # {"branch":"main","nodes":1,"workspace":"clean", …}
06 / 11

A real shell

Line editing, live schema, a prompt that knows where you are.

The old shell read raw lines: arrow keys emitted escape sequences into the parser and there was no history. It's now a proper REPL — and stays scriptable when piped.

EDITING

rustyline

History, recall and in-line editing; Ctrl-C cancels a statement, Ctrl-D exits. Interactive-only — piped input falls back to plain reads. (ADR-0033.)

IN-SHELL

Never leave to set up

:declare-label/-rel-type/-index, :commit, :status, :schema, :cancel — reusing the CLI's own code. Declared schema is picked up by the next query.

PROMPT

acetone:main*>

Branch-aware, with a * dirty marker. Tables align on display width (CJK/emoji), NULL is distinct from the string, errors go to stderr.

07 / 11

The review gate

A fresh reviewer on every PR — and it caught the one that mattered.

Every PR faced a fresh subagent with no implementation context, prompted to break it. It found real defects the implementer had missed.

#106 · BLOCKER

A no-flag acetone migrate would have silently rewritten all history to a chunk profile no init produces — its defaults were taken from the wrong constant. Now it preserves the repo's current params (a hash-preserving repair). Fixed and re-reviewed.

#104 · INJECTION

A fix reintroduced an unescaped label into a suggestion — the exact terminal-injection class the module guards against. Caught, reduced to a generic placeholder.

#109 · DATA LOSS

A piped statement with no trailing ; was silently dropped at EOF — a scripting foot-gun. Now flushed on EOF.

#108 · HARDENING

serde_json leaves DEL/C1 control bytes raw; the implementer pre-empted it with a residual-control escape pass so hostile values can't reach the terminal.

Decisions · security

Two ADRs, and a clean security gate.

ADRDecision · for ratification
0033Adopt rustyline for the shell — a runtime dep, acetone-cli-only, TTY-gated with a scriptable fallback (one transitive BSL-1.0 licence allowed)
0034Discover the repository from a subdirectory, preserving the isolated().with(Trust::Reduced) config-isolation boundary verbatim
MILESTONE SECURITY REVIEW · GATE READY

No blocker or high findings

Every terminal-control class (C0, C1, DEL, ANSI/ESC) is neutralised on every new output path; discovery preserves the isolation boundary; init can't alias an enclosing repo; no new unsafe; deps confined and cargo deny clean. One open risk (LOW), 7bn.19: bidirectional-format (Trojan-source) characters still pass the sentence-shaped sanitiser raw — pre-existing, visual only, not a 0.1.1 regression. Your call to pull in or defer.
