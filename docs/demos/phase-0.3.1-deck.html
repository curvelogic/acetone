03 / 12

The interesting part

Writing the manual found six real bugs.

Not code review — driving the tool to document it. Each was fixed in-phase, then folded back into the chapter that surfaced it.

  • Import bypassed constraints — a Service the manual imported without its required tier committed cleanly, where Cypher CREATE rejects it. A confirmed UNIQUE bypass too. 9gw · #184
  • Merge violations were invisible — a resolved merge left a dangling edge that only fsck could name; the conflict flow said nothing. jm8 · #187
  • Time travel to a tag didn't work — --at rejected short tag names and annotated tags that log already resolved. lqq · #185
  • log hid merged branches — first-parent only; needed git log --all to see the other side. b6q · #190
  • No --param, no branch start-point — capabilities the library had but the CLI couldn't reach. 9zt · #189, 7qw.1 · #190

Documentation-by-execution earns its keep. The verify-against-the-binary rule is now enforced in CI.

04 / 12

What shipped · hardening

Made safe against input you didn't write.

acetone-19x · #188

A real process-abort DoS, fixed

A query building a 200,000-deep value with reduce, then DISTINCT/ORDER BY, crashed the process with a stack overflow. Runtime value construction is now depth-capped; parameters bounded at ingestion.

acetone-c2a · #164

gc path traversal, killed

A crafted pack-sidecar stem could delete files outside the object directory — proven live, then closed. Plus co-tenant ref-ownership narrowed to an allow-list and a worktree-add TOCTOU closed.

acetone-0ds · #171

Terminal spoofing

Zero-width and invisible Unicode escaped on identifier output — including identifiers projected into result cells — completing the 0.1.1 bidi defence.

acetone-7vw · #172

Encoding guards

CBOR preallocation capped (memory amplification); non-round-trippable values rejected as node keys so identity can't diverge.

Executor budgets now charge var-length expansion, grouping, and replace() amplification #176; fsck peels tags and walks symrefs instead of aborting #165; migrate rewrites tags in a crash-recoverable swing #179.

05 / 12

Correctness · the enforcement story

A constraint now means the same thing on every path.

The manual found that import and put-node could persist a node CREATE would reject. The fix wasn't to patch two spots — it was to route every write surface through one shared checker, then audit the rest of the surface deliberately.

Write pathBefore 0.3.1Now
Cypher CREATE/SETenforcedenforced (unchanged)
importbypassed ✕check_nodes — atomic, names offenders
put-nodebypassed ✕check_upsert
declare-label retrofitsilent over bad datarefuses, names violating keys
merge completionenforcedenforced + violations named
fsckdidn't checkreports breaches as advisories

Two review cycles caught the put-node half — the same class as the import hole, found by insisting the plumbing audit close the class, not the instance.

What shipped · reachable through the tool

The library's capabilities, now on the CLI.

acetone-9zt · #189

query --param KEY=VALUE

Strict Cypher-literal binding — a bare word errors rather than silently becoming a string; no substitution into query text, so injection is structurally impossible. Composes with --at; shell :param.

acetone-b6q · #190

log --all + branch UX

Walks every branch tip; merge commits show a structural parents line that hostile message content cannot forge. branch NAME [REFSPEC] and branch --delete.

acetone-lqq · #185

Tag-aware time travel

--at v1.2 resolves short tag names and peels annotated tags, with git-parity precedence.

acetone-cbl.3 · #192

Errors that teach

Unknown label suggests declare-label; write-only queries drop the stray (no columns) line; map projections and blame arity get named messages.
