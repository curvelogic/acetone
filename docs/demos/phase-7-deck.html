The critical path

Fix the seams before the API freezes them.

The order was deliberate: govern first, then stabilise what the API will commit to, and only then expose it. Freezing a defect into a public signature is the one mistake 0.2 cannot take back.

1 · GOVERN

Resource governor

A deterministic work budget bounds every query — the blocker for any embedded or library use.

2 · STABILISE

Identity · values · seek

Stable relationship identity, a lossless value-domain contract, and a lazy store-backed IndexSeek — in parallel, each a thing the API will freeze.

3 · EXPOSE

Session + CLI façade

The library query entry point lands last, and the CLI is re-routed through it — so the API can't freeze the defects the first two steps removed.

03 / 13

The governed API · the headline

A query can't run away with the machine.

The 0.2 promise for embedders: hand acetone an arbitrary query and it returns, bounded, or declines — it never exhausts the host. A single deterministic work budget charges every row, expansion step, and allocation.

GOVERNOR · ADR-0036

One budget, charged everywhere

Time, result rows, expansion steps and collection/list size all draw on one budget. Var-length expansion, list-building functions and materialisation each charge it — no unbounded path.

SESSION · ADR-0039

The library entry point

A Session takes untrusted Cypher and runs it under the governor. This is the surface 0.2 freezes.

FAÇADE · ijq

The CLI runs through it

The CLI is now a client of the same governed path — so the library and the tool cannot drift apart in behaviour or bounds.

04 / 13

Frozen contracts

What the public API commits to.

Two things a stable API must never quietly change under a caller: which relationship you're looking at, and what a value means when it comes back. Both were fixed to a real contract before the freeze.

STABLE RELATIONSHIP IDENTITY · ADR-0037

Identity from the edge key, not a position

Relationships were addressed by a positional e{index} that shifted as the graph changed. Identity now derives from the edge key itself — the same relationship keeps the same id across queries and edits. Shares its root cause with the duplicate-edge fix.

VALUE-DOMAIN ROUND-TRIP · ADR-0038

A value reads back as itself

A typed value carrier threads temporal and Bytes values through the query adapter, so they return as their own domain — not stringified debug output. The acute symptom (temporal → string) is now structurally impossible.

05 / 13

Merge grows up

Two branches, different properties, no conflict.

Decision 4 of the spec promised per-property merge. Phase 7 delivers it — plus the lifecycle around it and the workflow it unlocks.

CELL-WISE MERGE · ADR-0035 · your call

Per-property, not per-node

Import sets os_version; a human sets owner. They no longer collide — the merge is at the cell, as the design record promised. A Greg-gated decision, taken at the last demo.

LIFECYCLE · ADR-0041

Abort, resolve, re-validate

Merge abort and mid-merge recovery, graph-violation resolution, and post-merge re-validation — the full lifecycle, not just the happy path.

CURATION · ADR-0042

Import vs human, reconciled

Re-importing no longer blats manual annotations: import lands on a branch and merges, reusing the cell-wise machinery.

CONFLICTS · s7d

Base / ours / theirs, side by side

acetone.conflicts now surfaces all three sides together, so a conflict is something you can actually read and resolve.

06 / 13

Faster and sturdier

Read less; survive more.

PERFORMANCE · ADR-0040

Lazy, store-backed IndexSeek

An indexed lookup touches only the matching rows in the store, instead of materialising a whole version per statement — mirroring the numeric cross-type and runtime-representation fixes so it never silently returns a subset.

ROBUSTNESS

The seams that used to crash

fsck runs on a repo with a damaged workspace manifest (zhp); a detached-HEAD linked worktree bootstraps instead of erroring (cm9); list-returning functions charge the collection cap (fab); an undeclared-label MATCH gets an advisory instead of a silent zero rows (7bn.5, ADR-0043).

Each one was a seam between the storage core and the runtime — exactly where 0.1's urgent defects clustered. Closing them is what makes acetone-core trustworthy enough to freeze.

07 / 13

Autonomous · Gate-D care

Two format-adjacent fixes, done to freeze standard.

The last two items you asked to be done autonomously — both touch durability or the on-disk format, so both got the full process: ADR, TDD, strongest-tier adversarial review.

DURABILITY · ADR-0044 · 7tf

A linked worktree survives a foreign git gc

A linked worktree's workspace ref lives under refs/worktree/*, which git does not enumerate as a gc root from the main worktree — so a stock git gc --prune=now could prune its uncommitted chunks. Now mirrored to a common-store anchor git always sees; acetone gc prunes stale anchors.

GOLDEN · ADR-0045 · 7bn.18

Pin the profile a real repo actually uses

The byte-exact goldens pinned only the 16 KiB test profile, not the shipped 64 KiB one. An empirical check showed the old dataset collapses to a single leaf at 64 KiB — so re-pinning would have lost coverage. Additively pinned both, with a cross-crate guard.

The review gate

A fresh reviewer on every PR — still earning its keep.

Every code PR faced a fresh subagent with no implementation context, prompted to break it. Across Phase 7 it kept catching real things — right through the final two PRs.

7tf · DURABILITY

The anchor was silently skipped for a worktree whose id wasn't valid UTF-8 — quietly dropping durability where the design fails loudly. Now it errors the save. APPROVE + CONFIRM.

7bn.18 · GATE-D PINS

The reviewer independently regenerated every pinned hash and framing byte under the shipped profile — byte-exact match — and swept the height plateau to prove 2000 entries isn't borderline. APPROVE.

SEAM FIXES

The governor, value carrier and IndexSeek PRs each drew findings on unbounded paths and subset-return edge cases — fixed before the API surface froze around them.

DISCIPLINE

Non-trivial fixes were re-reviewed; reviewers that stalled mid-stream were re-dispatched, never merged on a partial. The gate is reviewer sign-off, not CI.

09 / 13

Gate · security

The review found the one cap the governor missed.

A dedicated adversarial pass over the ~10k-line Phase 7 diff, aimed at the new attack surface. Almost everything held — and then it found a real hole in the exact property this phase freezes.

HIGH · FOUND

DISTINCT ran away from the governor

Both DISTINCT sites deduped with an O(n²) scan that charged the work budget nothing. UNWIND range(0,999999) RETURN DISTINCT x burned minutes of CPU while the odometer read ~3M — evading exit criterion #2, the guarantee being frozen at 0.2.

HIGH · FIXED

O(n) hash dedup, charged per row

A canonical value key (consistent with the existing equality) dedups in O(n) and charges the governor per kept row, so the caps now fire. Internal only — no change to the frozen Session API; TCK-neutral; fresh re-review brute-forced the equality and confirmed it.

Everything else verified sound: governor arithmetic (no overflow), the library path governed like the CLI, the value carrier, lazy reads on corrupt data, merge/persist, and the worktree-ref <id> — a validated single path component, not injectable.
