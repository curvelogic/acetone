Gate D · the freeze

Three audits, one silent gap, then frozen.

Three independent adversarial audits — key encodings, value CBOR, schema + prolly + chunking — each returned freeze-ready. All that stood between us and a stable format was one blind spot.

THE GAP THE AUDIT FOUND

The prolly format could drift silently

Every other artefact was byte-pinned; the node/chunker format was guarded only by self-relative tests that pass even when the bytes change. A format drift would have altered every root hash under an unchanged version. Closed by golden byte- and root-hash pins.

WHAT FREEZING MEANS

version 1, unchanged in bytes

The §3.4 encodings, the prolly node/chunker, and the manifest schema are all golden-anchored — a change flips a pin and fails loudly. The manifest's format_version is checked first on every decode, so a future-version repo is rejected, never misread.

04 / 11

The escape hatch

acetone migrate — the freeze made safe.

A freeze without a way forward is a trap. migrate re-encodes every version under a pluggable transform and rebuilds the commit graph with new hashes — preserving each commit's message, author and committer, identity and timestamp, verbatim.

ENGINE

Generic + faithful

Walk refs → topo-order → re-encode → remap parents → swing refs → reset workspace. Deterministic and idempotent; fsck stays clean.

TODAY

Re-chunk transform

Version-preserving, yet it rewrites every root and commit — exactly what exercises the full engine end to end.

DECISION

Engine now, demo at 0.2

The real cross-version transform slots into the same engine at the first format bump — no synthetic format invented to demo against.

FLAGGED FOR YOUR CALL

The schema index entry is frozen as {label, property}

Single-property, not speculatively widened for composite indexes. No scheduled index work needs a wider definition, and a half-designed widening risks freezing a worse format than a clean one migrate can evolve. Cheap to widen now if you'd rather — a migrate later if not.

The review gate

A fresh reviewer on every PR — and it earned its keep.

Every PR faced a fresh subagent with no implementation context, prompted to break it. Nearly every one caught something real.

#75 · MIGRATE

The engine handled merge commits, but it was unproven in-tree. Review demanded a two-parent migration test before merge.

#76 · PACKAGING

A real command injection via a crafted git tag name in the release workflow — which CI can never run. Closed with an env: var.

#74 · FREEZE

The prolly golden-pin gap, and an order-breaking §3.4 wording trap. The reviewer re-derived the git-blob hash and the datetime range by hand.

#73 · NAMESPACING

Missed tree paths in a CI script and a workspace ref — caught, then fixed before merge.
