# ADR-0031: Pre-0.1 spec-drift corrections

*Status: accepted — decided by ADR under the Autonomous Protocol (pre-0.1
hardening sprint); the deliberation record is the pre-0.1 review
(`docs/reports/pre-0.1-review.md`). Ratified by Greg at the pre-0.1 boundary review (2026-07-11) · Date: 2026-07-11 · Bead: acetone-do1*

## Context

The pre-0.1 review found several places where `docs/acetone-02-spec.md` (a
governing document) had drifted from what v0.1 actually ships. Per CLAUDE.md,
"when code and spec diverge, either fix the code or update the spec deliberately
— never silently," and a governing-document change is recorded as an ADR. These
are **corrections toward reality**, not design changes: the behaviour is fixed;
the spec is being made honest about it.

## Decision

Four corrections to `docs/acetone-02-spec.md`, all aligning the text with the
shipped 0.1:

1. **§1 positioning.** Sharpened from "a workbench … CLI and library" to the
   0.1 framing: a *solo, git-native workbench for a version-controlled asset
   registry* — imports become audited commits, `diff` becomes a change report,
   any git remote is backup/transport — and explicit that **the CLI is the 0.1
   product** while the `acetone-core` library API stabilises at 0.2.

2. **§7 CLI grammar.** Removed `push | pull | clone <url>` — acetone has **no**
   such subcommands; backup/transport is plain `git push`/`pull`/`clone` on the
   enclosing repository (§3.5 already said so). Removed `:diff` from the listed
   shell conveniences (the REPL has `:checkout`, `:log`, `:format`, `:quit`,
   `:help` — no `:diff`). Softened "the [library] is the real product surface"
   to "in v0.1 the CLI is the product surface; the library API is deliberately
   not frozen until 0.2 (gated on the query-engine resource governor)" — in both
   §7 **and** the §8 crate-layout line (which called `acetone-core` "the real
   product surface" and `acetone-cli` "the thin CLI client"), and downgraded §8's
   `acetone-cypher` "planner" to "heuristic planner" to match §5.3.

3. **§5.2 / §5.3 execution and time travel.** The §5.2 time-travel example used
   `AT 'main~5'`, a git *ancestry* refspec that the v0.1 resolver rejects
   (`resolve_commit` handles branch names, tags, full refs and hashes only);
   corrected to a resolvable refspec, with ancestry refspecs (`main~5`/`HEAD^`)
   noted as a planned convenience (`acetone-bvq`). The §5.3 execution model: The spec described "a physical plan of iterator-
   model operators" as reified structs. The v0.1 executor is a **clause pipeline
   over materialised row sets**; the named operators live as the pattern
   matcher's internals and clause handlers, not as standalone `Operator` structs.
   Planning is implicit/heuristic (the matcher's anchor choice); a costed planner
   and a lazy iterator model are deferred (store-backed `IndexSeek` is
   `acetone-cbl.11`). This matches the long-standing `run.rs` module note.

4. **§3.1 hash default.** Made the **SHA-1 default deliberate and explained**:
   the common hosting path (a plain git remote for backup/transport) still
   requires SHA-1 because major forges — including GitHub — do not yet host
   SHA-256 repositories; SHA-1's collision weakness is not a security boundary
   for content acetone addresses itself; `--object-format sha256` opts in; hash
   width is validated-not-assumed on every read.

## Consequences

- The spec no longer overstates the library ("real product surface") or lists
  commands and shell conveniences that do not exist, and it describes the actual
  execution model and the SHA-1 default honestly. No behaviour changed.
- The §5.3 iterator-model operators and the §7 library-API stabilisation remain
  the intended *direction* — Phase-7 items (`acetone-cbl.11` lazy seek,
  `acetone-vf6` library query API, `acetone-iq6` governor), not v0.1 claims.
- **Ancestry-refspec *support*** (resolving `main~5`/`HEAD^`) is deferred to
  `acetone-bvq`; only the misleading §5.2 *example* is fixed here, so the spec no
  longer shows a form the v0.1 binary rejects.
- **Not edited on a guess:** ADR-0017's status line ("agent decision, flagged for
  phase-boundary review") records an un-ratified decision; it is listed among the
  ADRs awaiting Greg's boundary ratification rather than rewritten. The roadmap's
  Phases 7–9 (`acetone-7ro`) are tracked separately.
